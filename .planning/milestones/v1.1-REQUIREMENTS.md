# Requirements: AI SEO Boost

**Defined:** 2026-04-20
**Milestone:** v1.1 — Interactive Guided Remediation
**Core Value:** Any website, pointed at this server, gets everything it needs to be recommended by ChatGPT, Claude, and Perplexity by name — with zero manual file editing.

## v1.1 Requirements

### Wizard Mode

- [x] **WIZ-01**: After `audit_ai_seo` completes, user is prompted to choose "Detailed report" or "Fix with wizard" — ✓ Phase 7, elicitation fork implemented
- [x] **WIZ-02**: `audit_ai_seo` accepts optional `businessContext` input alongside the existing `target` parameter — ✓ Phase 7, threads through to wizard envelope

### Issue Selection

- [x] **ISEL-01**: Wizard presents all audit issues as a toggleable checklist with severity/priority visible — ✓ Phase 8, dimension:status composite keys
- [x] **ISEL-02**: All issues default to selected — user deselects to exclude — ✓ Phase 8, schema default array pre-selects all
- [x] **ISEL-03**: User submits their selection to begin the fix sequence — ✓ Phase 8, Phase 8 envelope passed to Phase 9

### Context Handling

- [x] **CTX-01**: If business context provided upfront, it is reused across all tool invocations — ✓ Phase 9, seeds AccumulatedContext from businessContext
- [x] **CTX-02**: If no upfront context, wizard asks for required fields as each tool needs them — ✓ Phase 9, TOOL_FIELD_MAP lazy gap-fill
- [x] **CTX-03**: Context gathered mid-wizard is accumulated and never re-asked for subsequent tools — ✓ Phase 9, carry-forward proven by Scenarios G/H/I

### Tool Execution

- [x] **EXEC-01**: For each selected issue, wizard identifies and fires the correct fixing tool — ✓ Phase 10, switch dispatch on suggestedToolCall
- [x] **EXEC-02**: Tools execute sequentially in priority order — ✓ Phase 10, for...of selectedFindings loop
- [x] **EXEC-03**: Each tool asks only for fields not already in accumulated context ("fill in the blanks") — ✓ Phase 10, reads from AccumulatedContext
- [x] **EXEC-04**: Each completed tool run shows a confirmation of what was written or changed — ✓ Phase 10, per-tool elicitInput acknowledgment
- [x] **EXEC-05**: After all selected issues are resolved, wizard shows a summary of everything fixed — ✓ Phase 10, plain-text session summary

## Future Requirements

### Deferred from v1.0

- **LOC-01**: `generate_location_service_pages` full implementation (currently registered as stub)
- **CHAR-01**: iconv-lite charset detection — UTF-8-only is a documented v1 limitation
- **JS-01**: JS-rendered site support via headless browser for React/Vue SPAs without SSR

## Out of Scope

| Feature | Reason |
|---------|--------|
| Parallel tool execution in wizard | Sequential is safer — context accumulates, user can follow along |
| Undo / rollback wizard changes | Out of scope v1.1 — files can be manually reverted |
| Wizard state persistence across sessions | Single-run flow; no session storage needed |
| Batch wizard for multiple sites | Single-site focus consistent with v1 model |
