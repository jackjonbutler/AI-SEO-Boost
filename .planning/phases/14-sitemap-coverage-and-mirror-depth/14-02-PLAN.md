Run `npm run build` first to ensure `dist/` is current with the Phase 14 changes.

Then create `scripts/smoke-phase14-coverage.mjs`. The smoke script must work offline by monkey-patching `globalThis.fetch` before importing the module under test.

**Architecture:** The script patches `globalThis.fetch` to return controlled XML/HEAD responses, dynamically imports `checkMarkdownMirrors` from `../dist/audit/dimensions/markdown.js`, runs four scenarios, asserts on the returned `AuditFinding`, then restores `globalThis.fetch`.

**Pattern:** Follow the same pattern as `scripts/smoke-phase13-schema.mjs` — use a `let passed = 0; let failed = 0;` counter, log `[PASS]` / `[FAIL]` per scenario, and `process.exit(failed > 0 ? 1 : 0)` at the end.

**Four scenarios to implement:**

**Scenario 1 — Regular sitemap, 40 URLs, partial mirror coverage (COV-01)**
- Patch `fetch` to:
  - Return 200 with a `<urlset>` containing 40 `<url><loc>` entries (https://example.com/page-1 through /page-40)
  - For HEAD requests to URLs ending in `/index.md`: return 200 for the first 8 mirror probes, 404 for the rest (simulate 8/20 coverage)
- Call `checkMarkdownMirrors('https://example.com')`
- Assert: `finding.status === 'warning'` (8/20 = 40%, not 100% and not 0%)
- Assert: `finding.message` includes `'estimated'` (COV-03 label)
- Assert: `finding.message` includes `'/'` (showing N/M format like "8/20")
- Assert: `finding.message` does NOT include `'index.md'` as the checked URL (old binary pattern gone)

**Scenario 2 — Sitemap index file, child sitemap has 5 URLs (COV-02)**
- Patch `fetch` to:
  - First fetch (`/sitemap.xml`): return 200 with a `<sitemapindex>` containing one `<sitemap><loc>https://example.com/sitemap-posts.xml</loc></sitemap>` entry
  - Second fetch (`/sitemap-posts.xml`): return 200 with a `<urlset>` containing 5 `<url><loc>` entries
  - HEAD probes for `/index.md` variants: return 404 for all (0% coverage)
- Call `checkMarkdownMirrors('https://example.com')`
- Assert: `finding.status === 'fail'` (0% coverage)
- Assert: `finding.message` includes `'5'` or `'0/5'` — confirming 5 URLs were parsed (not 0)
- Assert: `finding.message` includes `'estimated'`

**Scenario 3 — Large sitemap (100 URLs), sample cap enforced (COV-03)**
- Patch `fetch` to:
  - `/sitemap.xml`: return 200 with `<urlset>` containing 100 `<url><loc>` entries
  - Track how many HEAD requests are made (increment a counter `let headProbeCount = 0`)
  - HEAD probes: increment counter, return 404
- Call `checkMarkdownMirrors('https://example.com')`
- Assert: `headProbeCount <= 20` (COV-03: no more than 20 probes)
- Assert: `finding.message` includes `'20'` or the message shows a denominator ≤ 20

**Scenario 4 — No sitemap (404 on /sitemap.xml), graceful fallback**
- Patch `fetch` to return a response with `status: 404` for the sitemap URL
- Call `checkMarkdownMirrors('https://example.com')`
- Assert: `finding.status === 'warning'`
- Assert: `finding.message` includes `'No sitemap'` or `'sitemap'` (describing the absence)
- Assert: No error thrown (the call must resolve, not reject)

**Implementation notes:**
- Each scenario must restore or re-set the fetch mock between runs — simplest approach is to set `globalThis.fetch` at the start of each scenario's async IIFE or use a shared `makeFetchMock()` factory
- The XML strings can be minimal — just enough tags to exercise the cheerio selectors
- Use `try/catch` around each scenario to catch unexpected throws and mark as FAIL
- Import path: `import { checkMarkdownMirrors } from '../dist/audit/dimensions/markdown.js'` — the `.js` extension is required (Node16 ESM convention used throughout this project)
