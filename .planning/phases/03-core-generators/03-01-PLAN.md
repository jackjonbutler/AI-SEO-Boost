All 5 dimension modules follow the same shape. Each exports a single async function that takes a `target: string` and returns `Promise<AuditFinding>`. None of them throws — any network/fs failure becomes a `status: 'warning'` finding with a message describing the issue. src/audit/dimensions/llms-txt.ts — `export async function checkLlmsTxt(target: string): Promise<AuditFinding>`:
- If isUrl(target): HEAD request to `new URL('/llms.txt', target).href`. Status 200 => `{ dimension: 'llms-txt', status: 'pass', severity: 'low', message: 'llms.txt found at site root' }`. Status 404 => `{ status: 'fail', severity: 'critical', message: 'llms.txt missing at site root', suggestedToolCall: 'generate_llms_txt' }`. Other status => warning.
- If local folder: `fs.access(path.join(target, 'llms.txt'))` — present => pass; ENOENT => fail with suggestedToolCall.
- Catch-all: return `status: 'warning', severity: 'medium'` with the error message.

src/audit/dimensions/robots-txt.ts — `export async function checkRobotsTxtAiAccess(target: string): Promise<AuditFinding>` plus `export const AI_BOTS = ['GPTBot', 'ClaudeBot', 'PerplexityBot', 'Google-Extended', 'CCBot']`:
- Helper inside this file: `function botAlreadyPresent(content: string, botName: string): boolean` using `new RegExp('^\\s*user-agent:\\s*' + botName + '\\s*$', 'im')` — case-insensitive per Google spec. Do NOT escape the bot name beyond what's in AI_BOTS (they are constant literals, no special regex chars).
- URL target: GET `<origin>/robots.txt` with AbortSignal.timeout(5000). Read body as text. Compute `missing = AI_BOTS.filter(b => !botAlreadyPresent(text, b))`. If res.status is 404 => fail with suggestedToolCall 'configure_robots_txt'. If missing.length === 0 => pass. Else => fail severity 'high' message `Missing AI crawler rules for: ${missing.join(', ')}` suggestedToolCall 'configure_robots_txt'.
- Local target: `fs.readFile(path.join(target, 'robots.txt'), 'utf-8')`. ENOENT => fail; otherwise same filter.
- Note: export AI_BOTS so 03-03 robots-txt generator (already built in Wave 2) can be checked for shared constant — if 03-03 already defined AI_BOTS in src/generators/files/robots-txt.ts, import and re-use that constant instead of duplicating. Check src/generators/files/robots-txt.ts first; if it exports AI_BOTS, do `export { AI_BOTS } from '../../generators/files/robots-txt.js'`. If not, define here.

src/audit/dimensions/schema.ts — `export async function checkSchemaMarkup(target: string): Promise<AuditFinding>`:
- URL target: fetch the HTML at target (GET, 5s timeout). Local target: read index.html from folder root if it exists; if no root index.html, walk dir for the first *.html file (use `fs.readdir(target, { recursive: true })` and filter, matching the 02-01 pattern).
- Load HTML with cheerio: `const $ = cheerio.load(html)`. Iterate `$('script[type="application/ld+json"]')` — for each element, `JSON.parse($(el).text())` inside try/catch. Collect `@type` values (handle both string and array per RESEARCH.md Code Examples).
- No JSON-LD present => `status: 'fail', severity: 'high', message: 'No JSON-LD schema markup detected', suggestedToolCall: 'generate_schema_markup'`.
- Some types present but missing LocalBusiness => `status: 'warning', severity: 'medium', message: 'Schema markup present but LocalBusiness not detected. Found: <types>', suggestedToolCall: 'generate_schema_markup'`.
- LocalBusiness present => `status: 'pass', severity: 'low', message: 'Schema markup found: <types joined>'`.

src/audit/dimensions/faq.ts — `export async function checkFaq(target: string): Promise<AuditFinding>`:
- Same HTML acquisition as schema.ts. Reuse a helper if you extract one — otherwise duplicate is fine for v1.
- Check 1: FAQPage JSON-LD — scan all JSON-LD scripts; if any has `@type === 'FAQPage'` => `status: 'pass', severity: 'low', message: 'FAQPage JSON-LD detected'`.
- Check 2: Heading heuristic — count `$('h1, h2, h3, h4').filter((_, el) => $(el).text().includes('?')).length`. If >= 3 => `status: 'warning', severity: 'medium', message: 'Question-style headings found (<n>) but no FAQPage JSON-LD', suggestedToolCall: 'generate_faq_content'`.
- Neither => `status: 'fail', severity: 'medium', message: 'No FAQ content detected', suggestedToolCall: 'generate_faq_content'`.

src/audit/dimensions/markdown.ts — `export async function checkMarkdownMirrors(target: string): Promise<AuditFinding>`:
- URL target: HEAD `<origin>/index.md` (the home page mirror). 200 => pass severity low message 'Markdown mirror detected at /index.md'. 404 => fail severity medium message 'No markdown mirror found for home page', suggestedToolCall 'generate_markdown_mirrors'.
- Local target: check for index.md or any *.md file in folder root (not recursive deep — one quick readdir). Present => pass; absent => fail with suggestedToolCall.

Every dimension MUST NOT throw. Wrap the whole body in try/catch and return a warning finding on unexpected errors. Use `.js` import suffix everywhere per Node16 ESM. Use `import * as cheerio from 'cheerio'` pattern matching existing 02-01 code.

Create src/audit/index.ts exporting `export async function runAudit(target: string): Promise<AuditReport>`:

1. Validate input: if target is empty string or not a non-empty string, throw `new Error('target must be a non-empty string')`. The orchestrator CAN throw — the tool handler catches it. Dimension checks MUST NOT throw; the orchestrator is allowed to.

2. For URL targets: resolve the origin. `const siteOrigin = new URL(target).origin` — so subsequent dimensions probe against the origin, not a deep path.
   For local targets: `await fs.access(target)` to confirm folder exists. If it doesn't, throw `new Error('Local target does not exist: <target>')`.
   Use the shared `isUrl()` helper from src/audit/types.ts (or wherever you put it in Task 1).

3. Run all 5 dimensions in parallel:
```ts
const findings = await Promise.all([
  checkLlmsTxt(probe),
  checkRobotsTxtAiAccess(probe),
  checkSchemaMarkup(probe),
  checkFaq(probe),
  checkMarkdownMirrors(probe),
]);
```
Where `probe` is `siteOrigin` for URLs and `target` for local paths. Because each check returns a finding rather than throwing, Promise.all will not reject.

4. Sort findings by severity descending (critical > high > medium > low):
```ts
const order: Record<Severity, number> = { critical: 0, high: 1, medium: 2, low: 3 };
findings.sort((a, b) => order[a.severity] - order[b.severity]);
```

5. Return `{ target, generatedAt: new Date().toISOString(), findings }`.

Import ordering: all dimension functions from './dimensions/*.js'. AuditReport/AuditFinding/Severity types from './types.js'. fs from 'node:fs/promises'.

```ts
async ({ target, businessContext: _businessContext }) => {
  try {
    if (!target || typeof target !== 'string' || target.trim().length === 0) {
      return {
        content: [{ type: 'text' as const, text: 'Error: target must be a non-empty string (URL or absolute local folder path)' }],

      };

    const msg = err instanceof Error ? err.message : String(err);
    return {
      content: [{ type: 'text' as const, text: `Error: ${msg}` }],
      isError: true,
    };
  }
}
```
