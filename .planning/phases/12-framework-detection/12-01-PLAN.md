Create NEW file src/audit/framework.ts implementing the pure detection function plus the I/O wrapper. Follow the research's signal map and confidence scoring exactly.

Structure (4 sections):

1. Imports — import * as cheerio from 'cheerio'; import type { FrameworkDetection, FrameworkConfidence } from './types.js'; import { isUrl } from './types.js';

2. FRAMEWORK_SIGNALS module-level constant — a Record&lt;string, { strong: Array&lt;(html, headers) => boolean&gt;, weak: Array&lt;(html, headers) => boolean&gt; }&gt; with 7 entries (Next.js, Nuxt, Astro, WordPress, Shopify, Hugo, Jekyll). Use the EXACT signal predicates from the research (see 12-RESEARCH.md "Framework Signal Map" section, lines 325-409). Every predicate takes (html: string, headers: Headers) and returns boolean. Use html.includes() for substring checks (not regex — per research Pitfall 2, minified HTML breaks exact attribute regex). Use cheerio.load(html) only inside the meta[name=generator] predicates for WordPress/Hugo/Jekyll.

3. detectFramework(html: string, headers: Headers): FrameworkDetection — pure function, NO I/O. Implement:
   - Iterate FRAMEWORK_SIGNALS entries, filter matching predicates, collect { strong, weak } counts per framework
   - If zero frameworks matched → return { name: null, confidence: 'none' }
   - Otherwise rank frameworks by (strong * 2 + weak) descending — tie-break in favor of the first entry (Object.entries order preserves insertion order)
   - For the top framework, totalSignals = strong + weak
   - If totalSignals >= 2 → confidence: 'high' (FWK-03 requirement)
   - Else if strong >= 1 → confidence: 'medium'
   - Else → confidence: 'low' (top framework had only 1 weak signal)
   - Return { name: topName, confidence }

4. fetchAndDetectFramework(target: string): Promise&lt;FrameworkDetection | null&gt; — I/O wrapper. Implement:
   - If !isUrl(target) → return null (local file-system target; no detection attempted)
   - try: fetch(target, { signal: AbortSignal.timeout(5000) }) — full GET (we need the body), no method override
   - Read res.text() for html
   - Return detectFramework(html, res.headers)
   - catch any error → return null (per research Pitfall 4: fetch failure must not break runAudit's Promise.all)

Critical constraints:
- File must use .js extension on the local import (project uses Node16 ESM — see 01-foundation patterns)
- No default export; named exports only
- detectFramework must be pure — no Date.now(), no fetch, no I/O
- fetchAndDetectFramework's outer try/catch MUST catch everything (including AbortSignal timeout errors) and return null — NEVER rethrow
- Do NOT create src/audit/schema-type-map.ts in this plan (Phase 13 owns that file per research anti-pattern note)

Reference the research's Code Examples section for the exact signal predicates — do not invent new ones. Only the 7 frameworks listed (Next.js, Nuxt, Astro, WordPress, Shopify, Hugo, Jekyll).
