- Cite a concrete fact from ctx (type, count, list, location, phone, website, description)

Required templates (the order below is the output order — it determines which pairs are kept when count < available):

**Identity (required fields — always fire — need ≥8 to guarantee minimum for sparse context):**
1. `What is ${businessName}?` → `${businessName} is a ${businessType}${location ? ' based in ' + location : ''}.`
2. `What type of business is ${businessName}?` → `${businessName} is a ${businessType}.`
3. `What industry does ${businessName} operate in?` → `${businessName} operates as a ${businessType}.`
4. `Is ${businessName} a ${businessType}?` → `Yes. ${businessName} is a ${businessType}${location ? ' serving ' + location : ''}.`
5. `Who is ${businessName}?` → `${businessName} is a ${businessType}${location ? ' located in ' + location : ''}${description ? '. ' + description : '.'}` (description only appended when present — guard with trim check)
6. `What kind of services does ${businessName} provide?` → if ctx.services && ctx.services.length > 0: `${businessName} provides ${services.length} service${services.length === 1 ? '' : 's'}: ${services.join(', ')}.` ELSE: `${businessName} provides ${businessType} services${location ? ' in ' + location : ''}.` (this template has NO null branch — it always fires because businessType is required; the branch chooses between two answer forms)
7. `Where can I learn more about ${businessName}?` → if ctx.website: `You can learn more about ${businessName} at ${website}.` ELSE: `${businessName} is a ${businessType}${location ? ' located in ' + location : ''}.` (always fires)
8. `What does ${businessName} do?` → `${businessName} operates as a ${businessType}${services && services.length > 0 ? ', offering ' + services.length + ' service' + (services.length === 1 ? '' : 's') : ''}${location ? ' in ' + location : ''}.` (always fires, enriched when optional fields present)

**Services (require ctx.services && services.length > 0 — return null otherwise):**
9. `How many services does ${businessName} offer?` → `${businessName} offers ${services.length} service${services.length === 1 ? '' : 's'}: ${services.join(', ')}.`
10. `Does ${businessName} offer ${services[0]}?` → `Yes. ${businessName} offers ${services[0]}${location ? ' in ' + location : ''}.` (use first service; null if services absent or empty)

**Location (requires ctx.location — return null otherwise):**
11. `What area does ${businessName} serve?` → `${businessName} serves ${location}.`
12. `Where is ${businessName} located?` → `${businessName} is located in ${location}.`

**Contact (requires ctx.phoneNumber — return null otherwise):**
13. `How do I contact ${businessName}?` → `You can contact ${businessName} at ${phoneNumber}${website ? ' or online at ' + website : ''}.`

**Website (requires ctx.website AND phoneNumber-branch not taken — return null when website absent):**
14. `Does ${businessName} have a website?` → `Yes. ${businessName}'s website is ${website}.`

For EVERY template accessing an optional field (location, services, phoneNumber, website, description), wrap in a guard that returns `null` when the required field is absent or an empty string after `.trim()`. Templates 1-8 must NEVER return null (they rely only on businessName + businessType, both required by BusinessContext) — this is what guarantees the ≥8 pair floor per RESEARCH.md Pitfall 1.

**Public function:**
```ts
export function buildFaqContent(ctx: BusinessContext, count?: number): FaqPair[] {
  if (!ctx.businessName || ctx.businessName.trim().length === 0) {
    throw new Error('businessContext.businessName is required for FAQ content');
  }
  if (!ctx.businessType || ctx.businessType.trim().length === 0) {
    throw new Error('businessContext.businessType is required for FAQ content');
  }

  const available = TEMPLATES
    .map((t) => t(ctx))
    .filter((p): p is FaqPair => p !== null);

  // Cap at count if provided; else cap at 10. Never exceed available pool.
  const target = Math.min(count ?? 10, available.length);
  return available.slice(0, target);
}
```

Constraints (RESEARCH.md Pitfalls + Anti-patterns):
- **No Zod import.** Handler validates at MCP boundary; pure function throws on structural errors (empty businessName/businessType) to make misuse loud — mirrors buildSchemaMarkup throw-then-handler-catches pattern (decision 04-03).
- **No fs import.** Pure function — no side effects.
- **No hedging language in ANY answer.** Re-read every template string for forbidden tokens before finalizing.
- **No invented facts.** Every value in an answer must come from `ctx`. No hardcoded numbers ("over 10 years"), no invented claims ("24/7 service"), no industry boilerplate.
- **No `import type` violations** — FaqPair is type-only; importing the runtime export would work but breaks the Pitfall 4 guideline.
- **Order matters.** TEMPLATES index 0-7 must ALL be always-fire (required-fields-only) templates. Index 8+ are conditional. This guarantees that for a minimal ctx (businessName + businessType only), at least 8 pairs are returned after filtering.
- **Answer-field guards must use `.trim()`** where the field is a string (location, phoneNumber, website, description) — Zod `.string().optional()` does not prevent empty strings.
- **Single trailing period** in every answer. No exclamation marks, no ellipses, no em-dashes.

Edge cases handled:
- `count` undefined → defaults to 10, clamped to available.length
- `count > available.length` → clamped to available.length (no padding, no duplicate pairs)
- `count < 8` with sparse ctx → returns whatever's available up to count; RESEARCH.md Open Question 2 — this is acceptable per the success criterion "returns 8–10 pairs" being the default (count=10) behavior, not a hard contract for lower counts. The handler does NOT treat < 8 as an error.
- Empty `ctx.services` array → service templates return null; identity templates still fire (all 8 guaranteed).
- `ctx.services.length === 1` → grammatical handling via `length === 1 ? '' : 's'` suffix.

```ts
async ({ businessContext, count }) => {
  try {
    if (
      !businessContext ||
      typeof businessContext.businessName !== 'string' ||
      businessContext.businessName.trim().length === 0
    ) {
      return {
        content: [{ type: 'text' as const, text: 'Error: businessContext.businessName is required' }],
        isError: true,
      };
    }
    if (
      typeof businessContext.businessType !== 'string' ||
      businessContext.businessType.trim().length === 0
    ) {
      return {
        content: [{ type: 'text' as const, text: 'Error: businessContext.businessType is required' }],
        isError: true,
      };
    }
    const pairs = buildFaqContent(businessContext, count);
    return {
      content: [{ type: 'text' as const, text: JSON.stringify(pairs, null, 2) }],
    };
  } catch (err) {
    const msg = err instanceof Error ? err.message : String(err);
    return {
      content: [{ type: 'text' as const, text: `Error: ${msg}` }],
      isError: true,
    };
  }
},
```
